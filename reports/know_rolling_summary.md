# Rolling Summary + Background Review（记忆系统分层治理）

> 口述稿 + 架构图
> 日期：2026-06-13
> 关联：mem0 全局摘要设计 + Learn Hermes Agent s20 + 工业级记忆治理

---

## 一句话总结

Rolling Summary 把记忆分成热路径（近期精确保留）和冷路径（远期分层压缩），Background Review 定期治理记忆膨胀和画像漂移。

---

## 口述稿（3 分钟版）

"Rolling Summary 解决的是上下文无限增长的问题。思路是分层：

**热路径**是近期记忆，比如最近 10 轮或当前 session 的对话，精确保留原始文本，查询时优先检索，确保用户刚说的'我喜欢川菜'不被压缩掉。

**冷路径**是远期记忆，按日→周→月分层摘要。每天把原始对话压缩成日摘要，保留关键事件和实体；每 7 天把日摘要合并成周摘要，保留重要决策；每 30 天把周摘要合并成月摘要，更新用户画像。

**分层合并的关键风险是丢失实体**。比如原始对话说'我女朋友叫小雨'，经过日→周→月压缩，月摘要可能只剩'用户有女朋友'，名字丢了。解决方案三招：① 保留各层原始数据，高层摘要丢失时可以穿透查下层；② 重要实体（人名、项目名）加白名单，摘要时不能去掉；③ 摘要完成后自检，检查原始对话中的关键实体是否还在。

**Background Review 和对话结束触发写入的区别**：
- 对话结束触发是'记录'——即时提取本次对话的新记忆，写入，增量处理
- Background Review 是'治理'——定期触发，处理全局记忆，压缩冗余、合并相似、更新画像、检查矛盾。有点像数据库的 vacuum，不是每次写入都做，而是定期整理。

两者缺一不可：只有即时写入，记忆无限膨胀，画像不更新；只有定期回顾，新记忆延迟生效。"

## 架构图（文字版）

### Rolling Summary 分层

```
当前对话（轮级，精确）
    ↓
热路径：最近 10 轮 / 当前 session（精确保留）
    ↓（超过 10 轮 / 换 session）
冷路径：
    ├── 日摘要（每天触发，保留关键事件+实体）
    ├── 周摘要（每 7 天合并，保留重要决策）
    └── 月摘要（每 30 天合并，保留长期趋势+用户画像）

查询时：热路径优先 + 冷路径按需穿透
```

### Background Review 流程

```
触发条件：每 50 轮对话 / 每天凌晨 / 记忆量超阈值
    ↓
阶段1：回顾近期记忆
    ├── 读取最近 N 轮新记忆
    ├── 提取可复用技能模式（如"用户总是优化交互模板"）
    └── 检查冗余记忆
    ↓
阶段2：压缩冗余
    ├── 合并相似记忆（3条"上下文优化"→1条综合）
    ├── 删除低重要性旧记忆（30天未访问 + 低importance）
    └── 更新时间衰减权重
    ↓
阶段3：更新用户画像
    ├── 从周/月摘要提取长期偏好
    ├── 更新 USER.md
    └── 检查画像漂移（矛盾项标记）
    ↓
阶段4：全局一致性检查
    └── 标记矛盾记忆，待用户确认
```

## 面试追问要点

| 追问 | 答案 |
|------|------|
| Q16：热路径和冷路径怎么分界？ | 按轮数（如最近10轮）、时间（如最近1小时）、或session（当前会话）。mem0 按 session 区分。 |
| 全局摘要过期/漂移怎么办？ | 三招：①保留原始数据可穿透；②重要实体加白名单；③摘要后自检。 |
| Background Review 和即时写入的区别？ | 治理 vs 记录，定期 vs 即时，全局 vs 增量。 |
| 长期运行怎么防止记忆膨胀？ | Background Review + 时间衰减 + 访问频率。 |
| 月摘要丢了'小雨'这个名字怎么办？ | 查询时穿透到日摘要，或自检时触发实体补全。 |

---

## 关键数字

- 热路径保留：最近 10-20 轮
- 日摘要周期：每天
- 周摘要周期：7 天
- 月摘要周期：30 天
- 压缩率：日 10:1 → 周 5:1 → 月 3:1
